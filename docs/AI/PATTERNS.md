## 3D空間計算

### パターン: 円柱カリング（Cylindrical Culling）

**問題**: 「円状」のカリングを実装したが、カメラ方向に追従しない

**原因**: 水平面（XZ平面）のみの距離判定では、カメラ→プレイヤー視線上のブロックを正しくカリングできない

**解決策**: 線分を中心軸とする円柱判定

```
1. 線分定義: 始点=カメラ位置, 終点=プレイヤー位置
2. ブロックから線分への最近接点を計算（投影）
   t = dot(block - lineStart, lineEnd - lineStart) / |lineVec|²
   closestPoint = lineStart + lineVec * t
3. tが0〜1の範囲内（線分上）か確認
4. ブロックと最近接点の水平距離を計算
   dist² = (block.x - closest.x)² + (block.z - closest.z)²
5. 半径以内ならカリング対象
```

**関連パターン**:
- 球状カリング: 中心点からの3D距離
- 円錐カリング: 角度ベクトルによる判定

## トラブルシューティング

### パターン1: カリングが効かない
**症状**: ブロックが非表示にならない
**確認事項**:
- [ ] 距離計算は距離の二乗で比較しているか（√計算を回避）
- [ ] 座標系は統一されているか（ブロック中心 vs ブロック隅）
- [ ] キャッシュが適切に無効化されているか

### パターン2: カリング範囲がおかしい
**症状**: 予想外のブロックが消える/残る
**確認事項**:
- [ ] 中心点（プレイヤー位置 vs カメラ位置）の定義を確認
- [ ] 2D（水平）と3D（空間）の距離計算を区別
- [ ] 「円状」=水平円 or 円柱 or 球かを明確化
